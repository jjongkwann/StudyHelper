# 프로세스 관리와 CPU 스케줄링

## 프로세스와 스레드의 차이

프로세스는 운영체제로부터 자원을 할당받는 작업의 단위이며, 스레드는 프로세스 내에서 실행되는 흐름의 단위이다.

- **프로세스**: 독립적인 메모리 공간(Code, Data, Stack, Heap)을 가짐
- **스레드**: 프로세스 내의 Stack만 별도로 할당, Code/Data/Heap은 공유
- 프로세스 간 통신(IPC)은 비용이 크지만, 스레드 간 통신은 공유 메모리로 가능

## 컨텍스트 스위칭

CPU가 하나의 프로세스에서 다른 프로세스로 전환할 때 발생하는 작업이다.

- PCB(Process Control Block)에 현재 상태 저장
- 새 프로세스의 PCB로부터 상태 복원
- **오버헤드**: 캐시 무효화, TLB 플러시, 레지스터 저장/복원
- 스레드 컨텍스트 스위칭은 같은 프로세스 내이므로 비용이 적음

## CPU 스케줄링 알고리즘

### 선점형 스케줄링
- **Round Robin**: 시간 할당량(Time Quantum)만큼 순환 실행
- **SRTF**: 남은 실행 시간이 가장 짧은 프로세스 우선
- **Multilevel Feedback Queue**: 여러 큐를 두고 우선순위 동적 조정

### 비선점형 스케줄링
- **FCFS**: 먼저 도착한 프로세스 먼저 실행 (Convoy Effect 문제)
- **SJF**: 실행 시간이 가장 짧은 프로세스 우선 (기아 문제)
